# Project Docs

Документы проекта хранятся в этом каталоге и классифицируются по назначению.

Базовое правило для всех артефактов проекта описано в [Artifact Taxonomy](operations/artifact-taxonomy.md).

## Классификация

- `product/` - продуктовые документы: product brief, ценностное предложение, аудитории, MVP, scope.
- `requirements/` - требования: функциональные, нефункциональные, пользовательские сценарии.
- `architecture/` - архитектура: системный дизайн, компоненты, интеграции, ADR.
- `planning/` - планирование: roadmap, декомпозиция задач, этапы запуска.
- `research/` - исследования: рынок, пользователи, конкуренты, регуляторика.
- `operations/` - операционные документы: процессы, роли, поддержка, качество.
